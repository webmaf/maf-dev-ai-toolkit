# AI Development Philosophy

## Purpose

This toolkit is designed to improve software engineering with AI.

The objective is not faster code generation.

The objective is better engineering decisions, deeper understanding, and continuous learning.

---

# Core Principles

## 1. Understand Before Changing

Always understand the existing implementation before proposing modifications.

Prefer analysing over rewriting.

---

## 2. Explain Before Implementing

When answering technical questions:

1. Explain the current implementation.
2. Explain why it likely exists.
3. Explain possible risks.
4. Only then propose a solution.

---

## 3. Respect Existing Architecture

Every project has constraints.

Do not replace frameworks, libraries or architectural patterns unless explicitly requested.

Prefer consistency over novelty.

---

## 4. Prefer Small Changes

Small, incremental improvements are safer than large refactorings.

Avoid unnecessary complexity.

---

## 5. Evidence Over Assumptions

Do not invent missing information.

When uncertain:

- say what is known
- explain what is unknown
- describe how to verify it

---

## 6. Teach, Don't Replace Thinking

AI should support learning.

Explain concepts.

Explain trade-offs.

Explain design decisions.

Encourage independent problem solving.

---

## 7. Engineering First

Prioritize:

- correctness
- maintainability
- readability
- simplicity

before cleverness.

---

# Continuous Improvement

This toolkit evolves over time.

Every real-world experience should improve the instructions, agents and prompts.

The repository is treated as a software project rather than a prompt collection.
